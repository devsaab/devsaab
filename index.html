Este es el primer curso de git

Cosas de interes, Este curso se basa en el uso de Git y de GitHub como gestores de los cambios en los diferentes proyectos que haremos a lo largo de nuestra carrera, este curso tambien explicara la diferencia entre Git y GitHub para las diferentes aplicaciones del mismo

El Git es un gestor de cambios para los proyectos; ¿Para que sirve esto? Cuando hacemos un proyecto nos pasa muchas veces que tenemos una carpata en la cual se encuentra el proyecto que estamos realizando, pero a lo largo que el proyecto avanza, las carpetas, archivos e imagenes se hacen mas complejos y extensos, eso sin contar el numero de versiones a trabajar la cual puede forma un desorden y hacernos la vida mas complicada al momento de encontrar un error, con Git, los cambios que hacemos quedan registrados, lo cual nos permite, guardar versiones del proyecto que en si mismas forman un programa, un ejemplo que es es muy comun es tener varias versiones de una aplicacion, entonces tenemos appv1, appv2, y appv3 cuando hacemos las pruebas en la ultima version nos damos cuenta que tiene un error que no se encontraba presente en la version anterior, con git podemos regresar a la version anterior para hacer las pruebas correspondientes para luego corregirlas en una proxima entrega, es algo asi como viajar en el tiempo. Esa es entre otras cosas una de las funcuines principales del Git las cuales explicaremos a lo largo de este curso Github GitHub, es una red social enfocada a los proyectos de desarrollo de programadores o coders, en esta red social podemos encontrar toda clase de proyectos desde su primera version de publicacion, hasta la ultima version y cada una con notas de los cambios, de problemas correcciones y toda la informacion relacionada al proyecto, con GitHub podremos compartir, leer y editar proyectos de todo tipo de codigo, desde lenguajes web hasta aplicaciones de escritori. ¿pero esto en que hace especial a GitHub? Con GitHub podemos gestionar nuestros paquetes a travez de Git, ¿Esto que quiere decir? Que al monmento de verificar los cambios de nuestros proyectos privados o compartidos podremos hacerlos a travez de una plataforma Online la cual nos permitira, aparte del respaldo de nuestro proyecto, brindar el apoyo y recibirlo a travez de la comunidad de programadores mas grande del mundos

  • Primer Paso: Descargar Git de su web oficial, Luego de la instalacion basica ejecutar el GitBash (Windows)
  • Comentarios: Este curso va de la mano con el uso de GitHub, para el siguiente paso se necesita como requisito tener una cuenta en GitHub
  • Segundo Paso: Luego de iniciar sesion en GitHub y de ubicarnos en la carpeta de nuestro proyecto inicial, hacemos uso de nuestro primer comando en Git, git ini "Este comando nos sirve para iniciar el Git"
